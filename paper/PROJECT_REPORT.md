# Project Report: Inference Complexity Scaling Laws in ANN-SNN Conversion

---

## 1. What This Project SHOULD Be (Original Spec)

A systematic empirical study that:
- Trains ANNs (VGG, ResNet) on 5 datasets
- Converts them to SNNs using SpikingJelly
- Measures accuracy, spikes, energy, latency at different timesteps
- Derives scaling laws: Acc(T) = a·log₂(T) + b, Spikes(T) = c·T + d, etc.
- Introduces novel metrics: SCI (Spike Complexity Index), CSI (Complexity Scaling Index)
- Produces publication-ready plots and statistical analysis
- Answers: "How does complexity scale with T, depth, and dataset difficulty?"

**Key deliverable:** 600 experiments (5 models × 5 datasets × 8 timesteps × 3 seeds) with REAL data from REAL trained models.

## 10. Critical Diagnosis: Why SNN Accuracy Is Low

### Real Data Pattern (VGG-9/MNIST):

| T | SNN Accuracy | Spikes/sample | Expected Acc |
|---|-------------|---------------|--------------|
| 1 | 10.28% | 7 | ~15-30% |
| 4 | 13.62% | 10,829 | ~50-60% |
| 16 | 9.75% | 96,005 | ~85-90% |

### Diagnosis:

**Spikes grow correctly** (roughly linear with T: 7 → 10K → 96K). This means the
neurons ARE firing. But accuracy doesn't improve → the spikes carry **wrong information**.

**Likely causes:**

1. **Threshold normalization failure:** SpikingJelly's `Converter(mode="max")` may set
   thresholds too high or too low for this architecture. With `mode="max"`, if there's
   one outlier activation, all thresholds get inflated → neurons barely fire (explains
   T=1 having only 7 spikes).

2. **AdaptiveAvgPool2d incompatibility:** SpikingJelly's converter may not handle
   `AdaptiveAvgPool2d(1,1)` correctly during torch.fx tracing. This could corrupt
   the spatial downsampling.

3. **Batch norm folding issue:** The converter folds BatchNorm into preceding conv layers.
   If the folding is incorrect, the converted weights are wrong.

4. **Output accumulation problem:** The evaluation accumulates SNN output over T steps.
   If the final layer doesn't spike properly (linear layer → no spikes), accumulation
   might not work as expected.

### Immediate Fix Actions:

```python
# FIX 1: Try percentile mode instead of max
converter = ann2snn.Converter(mode="99.9%", dataloader=calibration_loader)

# FIX 2: Use more calibration data
n_calib_batches = 32  # instead of 4

# FIX 3: Check if model needs modification for conversion
# Remove AdaptiveAvgPool2d, use manual pooling sized for input

# FIX 4: Test with simple model first (LeNet without adaptive pool)
# to isolate whether it's an architecture issue

# FIX 5: Try MUCH higher T (256, 512, 1024)
# Standard IF conversion in literature often needs T>500
```

### Key Insight:

Han et al. (CVPR 2020) report needing **2048 timesteps** for convergence with standard
IF neurons. Our testing only goes to T=16-64. The synthetic data assumed convergence at
T=64 (which was optimistic). **We may need T=128-512 for real convergence**, which would
dramatically change the scaling analysis.

**This is actually a scientific finding:** If standard conversion needs T>256 while
optimized methods (RMP, threshold balancing) achieve convergence at T~64, that
validates the need for novel conversion methods — which we can characterize
with our scaling framework.
